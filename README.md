# FitnessAPI

Backend API для платформы фитнес-тренировок и здоровья.

Проект развивается как серверная часть для:

- аутентификации пользователей;
- видео- и тренировочного контента;
- упражнений и тренировочных планов;
- блога о здоровье и комментариев;
- локализованной валидации и обработки ошибок.

## Стек

- Java 21
- Spring Boot 4
- Spring Security
- Spring Data JPA
- PostgreSQL
- Liquibase
- Gradle

## Реализовано

- регистрация пользователя;
- вход пользователя;
- выдача JWT access token;
- ротация refresh token с хранением сессий;
- профиль пользователя;
- CRUD API для видео;
- CRUD API для упражнений;
- CRUD API для тренировок;
- CRUD API для тегов тренировок;
- CRUD API для типов тренировок;
- добавление тегов, типов и упражнений в тренировку;
- удаление конкретного элемента тренировки по `trainingExerciseId`;
- избранные тренировки;
- рейтинги тренировок;
- сессии тренировок и выполнение упражнений внутри сессии;
- блог: статьи, категории и комментарии;
- Liquibase-миграции;
- локализованные сообщения валидации и ошибок.

## Основные модули

- `auth` - регистрация, логин, refresh flow и JWT;
- `video` - управление видео;
- `exercise` - управление упражнениями и их связью с видео;
- `training` - управление тренировками, избранным, рейтингами, сессиями и элементами тренировки;
- `tag` / `type` - классификация тренировок;
- `user` - профиль пользователя;
- `article`, `category`, `comment` - блоговый контент.

## API

- базовый servlet path: `/api/v1`;
- в проекте есть REST-эндпоинты для `videos`, `exercises`, `trainings`, `tags`, `types`, `articles`, `categories`, `comments`, `profiles`, `training-sessions`, `training-session-exercises`;
- реализованы сценарии `favourites` и `ratings` для тренировок;
- для тренировок доступны сценарии:
  - создание сессии: `POST /trainings/{trainingId}/sessions`;
  - список своих сессий: `GET /training-sessions`;
  - завершение/прерывание сессии: `POST /training-sessions/{sessionId}/complete`, `POST /training-sessions/{sessionId}/abandon`;
  - работа с упражнениями сессии: `GET /training-sessions/{sessionId}/exercises`, `POST /training-session-exercises/{sessionExerciseId}/start`, `.../complete`, `.../skip`.

## В планах

- трекинг прогресса;

- ролевая авторизация на уровне бизнес-операций.

## Запуск

1. Создать `.env` на основе `.env.example`.
2. Заполнить значения переменных окружения.
3. Подгрузить переменные в терминал или в конфигурацию IDE.
4. Убедиться, что PostgreSQL доступен и указанные credentials корректны.
5. Запустить приложение через IDE или командой:

```powershell
.\gradlew bootRun
```

Для проверки компиляции:

```powershell
.\gradlew compileJava
```

Шаблон переменных лежит в [.env.example](C:\Обучение\Программирование\Java\FitnessAPI\.env.example).

Важно: Spring Boot сам по себе не читает `.env` автоматически. Перед запуском переменные нужно экспортировать в окружение или подключить файл через конфигурацию IDE.
